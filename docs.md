# Telegram бот для получения информации по серийному номеру

Источником данных является таблица в google sheet. Первый столбец - серийные номера, следующие столбцы - данные.
Пользователь вводит в бот серийный номер, бот проверяет его на соотсветствие формату, после чего либо выдает сообщение о ошибке ввода, либо осуществляет поиск в первом столбце таблицы соответствующих данных. Если такой серийный номер не найден - выдает сообщение об ошибке. Если найден - выдает информацию из остальных столбцов.

## Пользовательский ввод
- `/start` выдает информацию о боте: что это и как пользоваться
- любая другая строка должна восприниматься как серийный номер и запускаться поиск.

## Вывод данных
Код не должен полагаться на определенную структуру таблицы, и выводить данные из всех столбцов таблицы. При этом для структрирования данных должны использоваться заголовки столбцов.
Пример таблицы:
| Серийный номер | Дата производства | Производитель | Модель | Комментарий |
| ------------- | ---------------- | ------------ | ----- | ----------- |
| 012345678912 | 2026-01-01 | Вася Иванов | Сатурн | Хороший |
| 012345678913 | 2026-01-01 | Петя Петров | Юпитер | Плохой |

Пример вывода данных для запроса `012345678912`:
```text
Дата производства: 2026-01-01
Производитель: Вася Иванов
Модель: Сатурн
Комментарий: Хороший
```

## Настройки бота (задаются в .env файле при создании бота)
- `BOT_TOKEN` - токен бота
- `SHEET_ID` - ID таблицы в google sheet
- `SHEET_NAME` - название листа в таблице
- `SHEET_PAT` - персональный токен для доступа к google sheet
- `SERIAL_NUMBER_COLUMN` - номер столбца, в котором находятся серийные номера
- `IGNORE_COLUMNS` - номера столбцов, которые не должны учитываться при выводе данных
